# Add New AEM EDS L4 Block Module

This workflow creates a new block module with all required files and configurations for the AEM EDS L4 project.
It follows the established pattern used by existing blocks (e.g., `faqs`, `effect-media`).

## Prerequisites

Before starting, collect the following information from the user:

| Parameter | Description | Example |
|-----------|-------------|---------|
| `BLOCK_NAME` | Kebab-case block name | `my-block` |
| `BLOCK_TITLE` | Human-readable title for UE | `My Block` |
| `BLOCK_ID_NUM` | Numeric ID from design spec | `125` |
| `BASIC_FIELDS` | Fields for the "basic" tab in model config | text, image, bgColor, etc. |
| `ADVANCED_FIELDS` | Fields for the "advanced" tab (optional) | font sizes, font color, etc. |

> **Naming Rules:**
> - `BLOCK_NAME` must be kebab-case (lowercase, hyphen-separated), e.g. `my-block`
> - `BLOCK_TITLE` is the display name in Universal Editor, e.g. `My Block`
> - The JSON file under `blocks/` must be prefixed with underscore: `_my-block.json`
>   (The glob pattern `../blocks/*/_*.json` only picks up files starting with `_`)

---

## Step 1: Create the block directory structure

Create the following files under `blocks/BLOCK_NAME/`:

```
blocks/BLOCK_NAME/
├── _BLOCK_NAME.json           # Block definition, models, and filters (MUST start with _)
├── BLOCK_NAME.js              # Block JavaScript entry point
├── BLOCK_NAME.scss            # Main SCSS entry (imports sub-files)
├── BLOCK_NAME.css             # Compiled CSS (generated by `npm run scss:build`)
├── modelConfig/
│   ├── _basic.json            # Basic tab fields
│   └── _advanced.json         # Advanced tab fields
└── styles/
    ├── _base.scss             # Base styles with RWD scaffolding
    ├── _product_line.scss     # Product-line-specific CSS variables
    └── _ue.scss               # Universal Editor specific styles
```

---

## Step 2: Create `blocks/BLOCK_NAME/_BLOCK_NAME.json`

This is the **most critical file**. It defines the block's component definition, model, and filter config.
The filename **MUST** start with `_` (underscore) to be picked up by the merge-json glob.

```json
{
  "definitions": [
    {
      "title": "BLOCK_TITLE (ID:BLOCK_ID_NUM)",
      "id": "BLOCK_NAME",
      "plugins": {
        "xwalk": {
          "page": {
            "resourceType": "core/franklin/components/block/v1/block",
            "template": {
              "name": "BLOCK_TITLE",
              "model": "BLOCK_NAME"
            }
          }
        }
      }
    }
  ],
  "models": [
    {
      "id": "BLOCK_NAME",
      "fields": [
        {
          "...": "./modelConfig/_basic.json#/fields"
        },
        {
          "...": "./modelConfig/_advanced.json#/fields"
        },
        {
          "...": "../../components/colorGroup/_colorGroup.json#/fields"
        },
        {
          "component": "text",
          "name": "L4TagNestedBlock",
          "value": "L4--nested-block--BLOCK_NAME",
          "hidden": true
        }
      ]
    }
  ],
  "filters": []
}
```

**Key points:**
- `definitions[].id` and `models[].id` must match `BLOCK_NAME`
- `template.name` is the display name in UE (use PascalCase / Title Case)
- `template.model` must match `models[].id`
- `L4TagNestedBlock` value follows pattern `L4--nested-block--BLOCK_NAME`
- `filters` is typically an empty array for simple blocks; populate it if the block has sub-items

## Step 3: Create `blocks/BLOCK_NAME/modelConfig/_basic.json`

Define basic configuration fields. Common field components:
- `tab` - Tab separator
- `text` - Text input
- `reference` - Asset reference (images)
- `select` - Dropdown select
- `boolean` - Toggle
- `richtext` - Rich text editor
- `container` - Grouped fields (with `collapsible: true`)

```json
{
  "fields": [
    {
      "component": "tab",
      "label": "basic",
      "name": "basic"
    },
    {
      "component": "text",
      "label": "Text",
      "name": "text",
      "valueType": "string",
      "description": "* Description for the author"
    }
  ]
}
```

---

## Step 4: Create `blocks/BLOCK_NAME/modelConfig/_advanced.json`

Define advanced configuration fields (font sizes, max widths, etc.):

```json
{
  "fields": [
    {
      "component": "tab",
      "label": "advanced",
      "name": "advanced"
    }
  ]
}
```

> **Note:** Adjust the field list based on the block's specific requirements. The above is a common template.

## Step 5: Create `blocks/BLOCK_NAME/BLOCK_NAME.js`

Block JavaScript entry point. Uses project utilities from `scripts/utils.js`.

```javascript
import {
  getBlockConfigs,
  getFieldValue,
  // getProductLine,
  isAuthorUe,
} from '../../scripts/utils.js';

// ── Constants ─────────────────────────────────────────────────
const DEFAULT_CONFIG = {};

// ── Entry point ───────────────────────────────────────────────

export default async function decorate(block) {
  try {
    const config = await getBlockConfigs(block, DEFAULT_CONFIG, 'BLOCK_NAME');
    const v = getFieldValue(config);
    const isUE = isAuthorUe();

    block.innerHTML = '';

    // --- Field values ---
    const colorGroup = v('colorGroup', 'text') || '';
    if (colorGroup) block.classList.add(colorGroup);

    // --- Layout ---
    if (isUE) {
      // UE-specific layout adjustments
    }

    block.innerHTML = `
      <div class="BLOCK_NAME-container">
      </div>
    `;

    // --- Animation ---
    if (!isUE) {
      // scroll animation setup
    }
  } catch (error) {
    // eslint-disable-next-line no-console
    console.error('Error decorating BLOCK_NAME: ', error);
    block.innerHTML = '<div class="error-message">Failed to load BLOCK_NAME block</div>';
  }
}
```

**Key patterns:**
- `getBlockConfigs(block, DEFAULT_CONFIG, 'BLOCK_NAME')` - Parses block fields using model definition
- `getFieldValue(config)` - Returns a helper `v(fieldName, 'text'|'html')` to access field values
- `isAuthorUe()` - Detects if running inside Universal Editor
- Always wrap in try/catch with error fallback HTML

## Step 7: Register the block in section filter

Add the block's `id` (BLOCK_NAME) to the `section` filter in `models/_section.json`.

Open `models/_section.json` and find the `"id": "section"` filter entry.
Add `"BLOCK_NAME"` to its `components` array:

```json
{
  "id": "section",
  "components": [
    "text",
    "image",
    "button",
    "title",
    "hero",
    ...existing blocks...,
    "BLOCK_NAME"
  ]
}
```

> **This step is essential!** Without it, the block won't appear in UE's Section "+" menu.
> The `component-definition.json` and `component-models.json` are auto-populated via the
> `../blocks/*/_*.json` glob, but the `section` filter in `_section.json` must be updated manually.

---

## Step 8: Build JSON and compile SCSS

Run the following commands to regenerate the merged JSON files and compile CSS:

// turbo
```bash
npm run build:json
```

// turbo
```bash
npm run scss:build
```

This will:
1. Merge `models/_component-definition.json` → `component-definition.json` (picks up `blocks/BLOCK_NAME/_BLOCK_NAME.json#/definitions`)
2. Merge `models/_component-models.json` → `component-models.json` (picks up `blocks/BLOCK_NAME/_BLOCK_NAME.json#/models`)
3. Merge `models/_component-filters.json` → `component-filters.json` (picks up `blocks/BLOCK_NAME/_BLOCK_NAME.json#/filters` + section filter from `_section.json`)
4. Compile `blocks/BLOCK_NAME/BLOCK_NAME.scss` → `blocks/BLOCK_NAME/BLOCK_NAME.css`

## Step 10: Push and verify in Universal Editor

1. Commit all new/changed files
2. Push to the remote branch
3. Either:
   - Use AEM Local Proxy (`npm run dev`) and enable the browser extension, OR
   - Merge to the branch that the UE environment points to (e.g., `dev`)
4. Hard-refresh the UE page (`Cmd + Shift + R`)
5. Click `+` inside a Section to verify the new block appears

---

## Common Pitfalls

| Issue | Cause | Fix |
|-------|-------|-----|
| Block not in `component-definition.json` after build | JSON filename doesn't start with `_` | Rename to `_BLOCK_NAME.json` |
| Block not in UE Section "+" menu | Not added to section filter | Add to `models/_section.json` filter `components` array |
| UE shows old block list | Browser cache / wrong branch | Hard-refresh or check AEM Local Proxy |
| SCSS compile error | Missing `@use '../../../styles/variable' as v;` | Add the import |
| Block JS error in console | Wrong `blockName` in `getBlockConfigs()` call | Must match `models[].id` exactly |

## File Relationship Diagram

```
models/_component-definition.json
  └── references: ../blocks/*/_*.json#/definitions
        └── blocks/BLOCK_NAME/_BLOCK_NAME.json → definitions[]

models/_component-models.json
  └── references: ../blocks/*/_*.json#/models
        └── blocks/BLOCK_NAME/_BLOCK_NAME.json → models[]

models/_component-filters.json
  └── references: ../blocks/*/_*.json#/filters
  └── references: ./_section.json#/filters
        └── models/_section.json → filters[] (section.components must include BLOCK_NAME)

blocks/BLOCK_NAME/_BLOCK_NAME.json
  └── models[].fields references:
        ├── ./modelConfig/_basic.json#/fields
        ├── ./modelConfig/_advanced.json#/fields
        └── ../../components/colorGroup/_colorGroup.json#/fields

npm run build:json (merge-json-cli)
  └── Generates:
        ├── component-definition.json  (from _component-definition.json)
        ├── component-models.json      (from _component-models.json)
        └── component-filters.json     (from _component-filters.json)
```
